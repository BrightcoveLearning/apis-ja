    クリップを管理する

    このトピックでは、CMS API を使用してビデオクリップを管理する方法について説明します。

    はじめに

    クリップとは、 Brightcove Social やその他のツールによって生成された他の動画のセグメントから作成された動画です。 クリップが Brightcove Social 経由で共有するために作成されている場合は、レンディションを削除してマネージドストレージを削減したり、動画ワークフローに表示されないように無効にしたりできます。

    クリップの検索

    CMS API には、アカウント内のクリップを見つけるのに役立つ特別な検索用語があります。

    • Q = %2BIS_クリップ:True -クリップのみを返します。
    • Q=%2DIS_Clip: True -非クリップのみを返します。
    • q=%2BIS_clip: false -非クリップのみを返します (機能的には前の項目と同じ)。
    • q=%2BClip_ソース_ビデオ_ID: video_id -指定したビデオから生成されたクリップを返します。
    注意事項:
    • %2B は URI エンコードされた + 記号です。
    • %2D はURIエンコードされた記号 です
    • 制限 と オフセット が適用されることを覚えておいてください。デフォルトでは、CMS API クエリは 10 個の動画 (最大:limit = 100) を返します。そのため、アカウント内にクリップが多数ある場合は、適切な 制限 と オフセット パラメータを使用して複数のリクエストを行い、すべてを見つける必要があります。 CMS APIカウントエンドポイントにGETリクエストを送信することで、すべてのクリップの数を取得できます。... /アカウント/アカウント_ID/ カウント /動画/ q=%2BIS_Clip:true

    アセットの削除

    ソーシャルメディアサイトへの共有以外の目的でクリップを使用しない場合は、アセットを削除して管理ストレージを減らすことができます。 ストレージの面では、レンディションが最も違いますが、必要に応じてポスターやサムネイルを削除することもできます（これはStudioのクリップを一般的なビデオではないと識別するのに役立ちます）。

    クリップ自体を削除 する必要はありません 。削除すると、ソーシャルアナリティクスを取得できなくなるからです。 また、クリップを非アクティブ化したり、将来の日付にスケジュールを設定したりしても、ソーシャル分析は取得 されません 。

    アセットを削除するには、 CMS API のアセットエンドポイント を使用します。 ロジックは簡単です。まず、アセットを取得して Video Cloud システムでその ID を取得してから、それらを削除します。

    通常、削除するポスターとサムネイルは1つだけですが、一般的に複数のレンディションがあるので、これをプログラムで行う場合は、GETレンディション要求によって返されたオブジェクトの配列をループし、レンディションを1つずつ削除する必要があります。 以下は、関連するAPIエンドポイントと、アセットを削除するタスクを実行するためのサンプルコード（JavaScript+ APIリクエストを作成するためにPHPで書かれたプロキシ）です。

    アセットエンドポイント

    アセットを削除する必要がある関連アセットエンドポイントを次に示します。 これらすべてのエンドポイントについて、ベース URL は次のようになります。

     https://cms.api.brightcove.com/v1/accounts

    レンディション

    GET レンディション

     /  アカウントID  /動画/  動画_id  /アセット/レンディション

    レンディションの削除

     /  account_id  /videos/ video_id  /アセット/レンディション/ rendition_id

    ポスター

    GETポスター

     /  account_id  /videos/ video_id  /アセット/ポスター

    ポスターを削除する

     /  account_id  /videos/  video_id  /アセット/ ポスター/ poster_id

    サムネイル

    サムネイルを取得する

     /  account_id  /videos/ video_id  /assets/サムネイル

    サムネイルを削除する

     /  account_id  /videos/  video_id  /assets/thumbnail/  thumbnail_id
