入門

In-Page Experience API を使用すると、ページ内エクスペリエンスをプログラムで管理できるため、CMS 統合に特に役立ちます。これにより、ユーザーは Studio の [ギャラリー] に移動するのではなく、自分の CMS から管理エクスペリエンスを作成することができます。

このクイックスタートでは、ユーザーは次のことを可能にする実装例を提供します。

  1. アカウントのすべてのページ内エクスペリエンステンプレートを取得する
  2. 選択したテンプレートから基本的なエクスペリエンスを作成する
  3. プレイリストまたは動画のリストを選択して、エクスペリエンスに追加します
  4. 選択した動画/プレイリストでエクスペリエンスを更新する
  5. エクスペリエンスを公開する

以下に、アプリロジックの概要を示します。

アプリ

このアプリのすべてのコードは、この GitHup リポジトリにあります。

ペンのクイックスタートを参照してください。インページエクスペリエンスプラットフォームAPI Brightcove Learning Services（ @ rcrooks1969） オンCodePen。

CodePen を使用する

上記のCodePenを効果的に使用するためのヒントは次のとおりです。

  • [ 結果 ] ボタンをクリックして、プレーヤーの実際の表示を切り替えます。
  • HTML/CSS/JS ボタンをクリックして、コードタイプの 1 つを表示します。
  • 右上隅にある CodePen の編集をクリックして、この CodePen を自分のアカウントにフォークします。
  • このサンプルに関連付けられているすべてのコードを、この GitHub リポジトリで検索します。

プロキシコード

このページのサンプルアプリを独自のバージョンで作成するには、独自のプロキシを作成してホストする必要があります。（Brightcoveラーニングサービスで使用されるプロキシは、Brightcoveのドメインからのリクエストのみを受け付けます）。プロキシコードの 2 つのバージョンをダウンロードできます。

  • これは、クライアント資格情報が要求とともに渡されることを期待する一般的なバージョンです
  • このバージョンでは、クライアント資格情報をプロキシ自体に 25 ～ 26 行目に保存できます (推奨)
