    よくある質問:アナリティクス API

    以下は、アナリティクス API に関するよくある質問です。 Video Cloud Studio 分析モジュールに関してよく寄せられる質問の回答については、「 FAQ: 分析モジュール 」を参照してください。

    日付範囲

    特定の日のデータを取得するにはどうすればよいですか？
    開始値と終了値の両方 を、 その日付に設定します。 例: from=2013-05-12&to=2013-05-12 。 また、日付ディメンションでクエリを実行することもできます。 日付 ディメンション は、他のディメンションと組み合わせることができます。
    Analytics モジュールと Analytics API は、同じ日付範囲の識別データを提供する必要がありますか？
    はい。ただし、比較的新しいデータを比較し、StudioとAPIからデータを異なるタイミングで取得する場合は、異なる場合があります。
    1. データが 3 日未満です。 3 日未満の分析データは暫定的なものであり、いつでも変更される可能性があります。
    2. 同じ期間のデータは、作成日が 32 日未満だったときと、作成日が 32 日後に再び参照されます。 32 日後、データは別の「履歴」リポジトリに移動され、その時点で一部のデータ、特に詳細なエンゲージメントデータを破棄します。 つまり、エンゲージメントスコアなどの計算項目は、より粒度の低いデータに対して計算が行われるため、わずかに変化する可能性があります。

    データの細分性

    最近のアナリティクスデータの細分性はどのくらいですか？
    最新の 32 日間（現在の日を含む）では、値が 1 時間単位のバケットに保存されるため、Analytics API は 1 時間の粒度をレポートします。 ただし、現在の時間はまだいっぱいになっているため、現在の時間に該当するデータを要求すると、API に 1 時間よりも詳細な粒度を持つように見えます。

    たとえば、次のようになります。

    1. 3 時間前から 9:15 と 9:20 のデータを要求した場合、同じ時間バケットに含まれる場合は同じ値が得られることがあります。ただし、すべての分析データは暫定的であり、リコンサイルされるまで変更される可能性があるため、異なる場合があります。
    2. 10分前のデータを要求し、5分待ってから再度尋ねると、同じバケットにあっても別の値を取得する可能性があります。これは、バケットがまだ更新中であるためです。
    分析データの更新間隔はどのくらいですか。
    /data エンドポイントは、現在、クエリの間隔が 5 分間キャッシュされているため、過去 1 時間のトラフィックを含む動画のリストでは、5 分間隔が最小の遅延で、動作のデルタが得られます。
    アナリティクスの履歴データの細分性はどのくらいですか？
    過去 32 日よりも前の日付の場合、Analytics API は完全な日の値を報告します。 これは、あなたが要求した場合 = 1368334306919&to=1378446336919 （日、2013年5月12日（日）04:51:46 GMTから2013年9月6日 05:45:36 GMT）、あなたが =2013-05-12&to=2013-09-06 から要求した場合と同じ結果を得ることを意味します

    アイテムとフィールド

    1 つ以上の動画について、ある期間にストリーム開始回数を示す最良の方法は何ですか？
    video_view メトリックを見てください。これは常にストリームの開始と等しくなります。 この指標はすべてのレポートで確認できます。サマリーには、照会した日付範囲内のすべての動画のストリーミング開始が表示されます。 アカウント内のすべての動画のストリーム開始を確認するには、次のリクエストを実行します。
     https://analytics.api.brightcove.com/v1/data?accounts={account_ids}&dimensions=video&limit=all&offset=0&fields=all
    レポート 内のすべての商品を返品するにはどうすればよいですか？
    レポート の limit パラメータを all に設定します。
    アイテムに対してデフォルトで返されるデータ フィールド は何ですか。
    デフォルトでは、 video_view と、要求されたディメンションに対応するフィールド ( destination_domain など) のみが返されます。 追加のフィールドを取得するには、 fields = field_name1、field_name2 を設定して、選択したフィールドを返します。
    フィールド値が null の場合はどういう意味ですか？
    データフィールドの NULL 値は、要求されたデータが処理されていないことを示します。 最も可能性の高い理由は次のとおりです。
    • リクエストされたデータは最新のものであり、まだ処理されていません
    • リクエストされたデータは非常に古く、現在の分析システムにまだインポートされていません
    いずれかの動画の名前を変更しました。Analytics データに古い名前が表示されるのはなぜですか。
    新しい名前は、新しい分析イベントに対して記録されますが、履歴データの名前は変更されません。返されるビデオ名は、視聴時の名前になります。
    エンゲージメント値の整数が常に表示されないのはなぜですか？
    エンゲージメント番号が小数点で返されることがあります。 理由は、エンゲージメントが正規化されているためです。つまり、「video_percent_viewed *（video_engagement_25/video_engagement_sum）」の比率であるため、クエリに選択された時間範囲に基づいて、正確に分割されない場合に浮動小数点数が表示されることがあります。
    プレーヤーで結果をフィルタリングするにはどうすればよいですか？

    プレイヤーの w here フィルターを使用する場合は、次の点に注意してください。

    • ブライトコーブのプレイヤー -プレイヤー URL を次の形式で入力します。 players.brightcove.com/アカウント ID /プレイヤー ID _default
    • スマートプレイヤー-プレイヤーIDを入力してください

    レポート

    プレーヤーが使用している帯域幅を見ると 0 GB と表示されるのはなぜですか？
    これまで、ブライトコーブはプレーヤー別の帯域幅を報告してきました。そのため、プレーヤーごとに分類したいお客様には、これらのメトリックを帯域幅コストを割り当てる手段として使用できます。 しかし、マニフェスト駆動型の配信（今日のHLSと将来はDASH）に移行するにつれて、セグメント化されたビデオの性質により、プレーヤーによるトラッキング帯域幅が不正確になります。 したがって、使用状況レポートでは、プレーヤーのディメンションごとの帯域幅は廃止されます。 したがって、今後は、パフォーマンスレポートでプレーヤーが閲覧した秒数を使用して、プレーヤーごとにコストを配分することをお勧めします。

    雑多な

    分析レポートから従業員のビューを除外できますか？
    これを行う簡単な方法はありません。トラフィックの多い動画では、従業員の視聴回数はおそらくごくわずかで、分析への影響はごくわずかです。 しかし、これを行うことが重要だと思うトラフィックの少ない動画の場合、最も簡単な解決策は、プロダクションプレーヤーを複製し、従業員にそのコピーのビデオをテスト/表示させることです。 フィルタを使用して、Analytics モジュールのカスタムレポート機能または Analytics API のいずれかを使用して、実動プレーヤーでのみレポートを作成できます。
