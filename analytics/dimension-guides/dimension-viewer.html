視聴者レベルの分析は、すべてのアカウントで利用できるわけではありません。アカウントでこの機能をオンにすることに興味がある場合は、アカウントマネージャーに連絡してください。

URL パラメータ

分析 API レポートでは、次の URL パラメータがサポートされています。

URL パラメータ
パラメーター 説明 必須 値 デフォルト
account 報告したいアカウント はい 1 つ以上のアカウント ID をカンマで区切ったリスト なし
dimensions レポートするディメンション はい 1つ以上の次元をカンマで区切ったリスト (一部の組合せは無効です) なし
where レポートのフィルタを指定するために使用します いいえ {dimension} == {value} -セミコロンで区切られたリストとして1つ以上 なし
limit 返品するアイテムの数 いいえ 正の整数 10
offset スキップする項目の数 いいえ 正の整数 0
sort アイテムを並べ替えるフィールド いいえ リクエストの有効なフィールド Viewer_view
fields 返すフィールド いいえ レポートするディメンションによって異なります-を参照してください田畑利用可能なフィールドについては、以下をご覧ください。 Viewer_view
format 結果を返す形式 いいえ json | csv | xlxs json
from リクエストの日付範囲の開始日 いいえ ISO 8601 日付（YYYY-MM-DD）、エポック時間（ミリ秒）、文字列 alltime、または相対日付（-1m）。エンゲージメントエンドポイントでは、過去 32 日以内の日付のみが許可されるか、または recalciled=false の場合も許可されます。 今までの30日前

使用可能なフィールド

ディメンションの次のフィールドを返すことができます。

  • viewer -イベントビューア（SSO IDまたはデバイスID、またはユーザーエージェントとIPに基づく一意の文字列）
  • video_view -視聴者の再生が開始された回数
  • time -ミリ秒単位のビデオビューのエポック時間
  • engagement_score - エンゲージメントスコア

フィルタ値

フィルタとして使用すると、ディメンションを使用して結果を絞り込むことができます。

フィルタ値：カンマ区切りリストまたはviewer.q ==としてのビューアID {viewer field}：{value}
