    CMS/再生API: 動画検索v1

    このトピックでは、CMS API を使用してアカウントで動画を検索する方法について説明します。 CMS API は、ビデオクラウドライブラリ内のビデオを検索するためのプログラム的な方法を提供します。 このトピックでは、検索構文について詳しく説明します。 注：これは元の検索構文です。より単純な ビデオ検索構文 v2 を使用することをお勧めします 。

    はじめに

    このトピックでは、次の操作を行う方法を学習します。

    • q パラメータを使用して基本検索を作成および制限する
    • 検索結果の並べ替え
    • 必須語と除外語を使用した検索
    • 引用符で囲まれた検索を使用して、用語と複数の単語を一致させる
    • ユーザー設定フィールドの検索
    • 特定の日付および範囲を持つ日付フィールドを検索する
    • 検索条件を組み合わせる
    このトピックでは、ビデオ検索について説明します。「 プレイリスト検索 」も参照してください。

    API の使用状況

    検索機能は、CMS API または再生 API のいずれかで使用できます。

    どちらの API の検索文字列には、少なくとも 3 文字が含まれている必要があります。

    CMS API

    CMS API で検索を使用する場合は、以下が適用されます。

    • すべてのリクエスト（検索を含む）には、アクセストークンを含む認証ヘッダーが必要です。 クライアントの認証情報を取得し、それらを使用してアクセストークンを取得する方法の詳細については、「 Brightcove OAuth の概要 」を参照してください。
    • 検索から返される動画の最大数に制限はありませんが、レート制限が適用されます。
    • 検索結果には、再生可能かどうかに関わらず、アカウント内のすべての動画や、地域制限があります。

    API リクエスト/レスポンスの詳細については、『 CMS API リファレンス 』の「 ビデオの取得 」セクションを参照してください。

    再生API

    再生 API で検索を使用する場合は、次のことが適用されます。

    • Playback API を使用した検索リクエストには、 検索が有効な ポリシーキーが必要です。
    • 検索から返される動画の最大数には 制限 があります。
    • 検索結果には、再生可能な動画のみが返されます（ 状態:非アクティブ の動画は無視されます）。
    • 検索では、結果から地域制限付き動画を除外するなど、再生ポリシーの制限が適用されます。
    • 結果をキャッシュすると、リクエスト率が高くなり、レスポンスが速くなり、レート制限はありません。
    再生 API の検索では、リクエストあたり 100 動画に制限があり、上限は 1000 動画です。 検索結果のページ表示方法を 説明します。 1000 を超える動画を返す必要がある場合は、CMS API を使用する必要があります。

    API リクエスト/レスポンスの詳細については、「 再生 API リファレンス 」の「 ビデオの取得 」セクションを参照してください。

    基本検索

    メディアライブラリ内の用語を検索するには、 q パラメータを使用します。

     q= {検索用語}

    指定する検索用語は、スペースで区切られた URL エンコードされた用語リストである必要があります。

    検索では ステミング がサポートされます。 ステミングとは、単語のルートと、同じルートに由来する他の単語に単語をマッピングすることです。 たとえば、「run」で検索すると、指定したフィールドに「run」または「run」が含まれる動画と一致する必要があります。 「ルーン」にはルートとして「走る」がないため、「ルーン」には一致 しません 。

    q=bird などの検索語に修飾子を指定しない場合、リクエストは次のフィールドでその値が検索されます。

    • id [1-1]
    • 名前
    • 説明
    • description
    • テキスト （実際のメタデータフィールドではなく、 名前 、 説明 、および long_d escriptionを検索するために使用できる擬似フィールド（例： q=text：bird ）
    • タグ
    • reference
    • custom_fields (すべてのカスタムフィールドを検索)
    • custom_field_name (特定の名前付きユーザー設定フィールドを検索) [1-2]

    注意事項

    [ 1-1] 注：一貫性のために idによる検索が含まれていますが、q=id: 12345 で検索すると、リクエストとまったく同じ結果が返されます https://cms.api.brightcove.com/v1/accounts/{account_id}/videos/12345
    [ 1-2] リストタイプのカスタムフィールドがあり、いくつかの値のうちの1つを持つビデオを返す場合は、次のようにします。

    たとえば、 赤 、 緑 、 黄 、または 青 の値を取ることができる 色 というフィールドがあるとします。 そのフィールドに 緑 または 青 の値が設定されているビデオを見つけたいとします。

     q=カラー：グリーン%20カラー：ブルー。

    例：このリクエストは、上記のフィールドの少なくとも 1 つに bird の値を持つ動画を返します。

     https://cms.api.brightcove.com/v1/accounts/921483702001/videos?q=bird
    3文字未満の検索文字列は機能しません。代わりに、すべてのビデオが返されます。

    検索を特定のプロパティに制限する

    q=name: bird などの検索語句の修飾子を指定すると、リクエストはビデオ 名 フィールドから bird の値を検索します。

    例:このリクエストは、 名前 フィールドに 野生 動物の価値がある動画を返します。

     https://cms.api.brightcove.com/v1/accounts/921483702001/videos?q=name:wildlife

    検索結果のソート

    s ort パラメータを使用すると、ビデオの Get リクエストの結果を並べ替えることができます。 次の項目でソートできます。

    • reference
    • 名前
    • created
    • published
    • updated_at
    • starts_at
    • schedule.ends
    • 状態
    • total
    • plays

    ソートを使用して結果を明示的に ソート しない場合、結果は、用語頻度/逆文書頻度または TF-IDF と呼ばれるアルゴリズムに従ってソートされます。 詳細 はこちら をご覧ください。

    たとえば、 Coastal、city という用語で検索し、動画メタデータのどこかにその用語（ 名前 、 説明 、 タグ など）があり、検索結果の並べ替え条件（例では、それらはすべて同じ schedule_starts_at 日付/時刻を持ちます）。 動画の表示結果の高さは、一方または両方の用語がメタデータに表示される頻度によって決まります。また、動画ライブラリ全体で最も頻繁に表示される用語に重みが大きくなります。

    必須/除外された用語

    検索語句を必須として（返された動画は一致しない）、除外する（返された動画は一致しない）ことができます。 これは、URI エンコードされた + (%2B) または - 記号で制御されます。

    必要な用語を示すために + を使用する場合は 、%2B としてエンコードする必要があります。

    必須/除外条件
    例 URL エンコード 意味
    +foo Bfoo 動画には、 名前 、 説明、 long_d escription 、 タグ 、 参照ID、 カスタムフィールドに f oo という用語を含める必要があります
    +カスタムフィールド:foo %2BCustom_fields: foo ビデオには、いくつかのカスタムフィールドの値 foo を含める必要があります
    +foo-bar %2bfoo%20-bar 動画には foo という用語を含める必要がありますが、 名前 、 説明、long_description 、 タグ 、reference_ id 、 カスタムフィールド には b ar という用語を含めることはできません。
    +名前:foo-名前:バー %2bname: foo%20-name: バー 動画には foo という用語を含める必要がありますが、 名前 に bar という用語を含めることはできません。

     https://cms.api.brightcove.com/v1/accounts/921483702001/videos?q=%2Btags:sea%20-tags:lake

    他のパラメータと組み合わせる

    検索（ q パラメータを使用）は、 ソート 、 制限 、 オフセット などの他のパラメータと組み合わせることができます。 すべての URL パラメータは & で区切られます。 パラメータの順序は関係ありません。

    検索パラメータ 制限 の最大値は、リクエストごとに 100 です。 動画 ページネーションについては、検索結果 ページをご覧ください。

    例

    例：このリクエストは、 タグ フィールドに bar の値を持たなければならず、値 foo を含む 名前 を持つ可能性のある動画を返します。

     ... /動画？ q=名前:foo% 20% 2Btags: bar&Sort=updated_at

    例:このリクエストは上記と同じ動画を返しますが、さらに updated_at フィールドで結果をソートし、その結果を 10 動画に制限します。

     ... /動画？ ソート=updated_at&q=名前:foo% 20% 2Btags: bar&limit=10

    引用された検索用語

    デフォルトでは、検索語句と類似する語句が検索語句と一致します。 複数の単語をマッチさせたい場合は、単語を引用符で囲みます。

    ほとんどのブラウザやその他のエージェントは、リテラル引用符（「...」 ) が正しく検索されますが、引用符で囲まれた検索語句が正しい結果を返すように見えない場合は、引用符を%22 % 22..% 22 に置き換えてみてください) 。

      
q= "foo」か q=%22foo%22 
 q= "foo%20bar」か q=%22foo%20bar%22
    引用マークには、一致する動画にすべての用語が含まれている必要がありますが、必ずしもその用語に完全に一致するとは限りません。 たとえば、「ホームラン」 で検索すると、タイトルに homeless and run ning という単語が含まれる動画が返されることがあります。

    これは、特定のフィールドを検索するときにも機能します。

      
q=名前:"ホーム」
 q=名前:"ホーム%20ラン」

    複数の単語

    例：このリクエストは、 タグ フィールドに 海 または 哺乳動物の いずれかの値を持つ動画を返すことに注意してください。

     https://cms.api.brightcove.com/v1/accounts/921483702001/videos?q=tags:sea,mammal

    しかし, 次のリクエストは、タグを持つ動画のみを返します 海, 哺乳動物 .

     https://cms.api.brightcove.com/v1/accounts/921483702001/videos?q=tags:"sea,mammal "
    タグは、複数のタグをカンマ区切りリストとして検索できる特別なケースです。名前、説明などの他のフィールドには当てはまりません。複数の用語については、次のように + 記号で用語を組み合わせます。 q=tags："bird" %20name："owl" 。

    カスタムフィールド

    動画用に定義した任意のカスタムフィールドで検索できます。

     q=my_field: foo 
 q=my_field: "foo」

    注：すべてのカスタムフィールド値は文字列として扱われます。 たとえば、値 true または falseを取ることができるリストタイプのカスタムフィールドがある場合、se archはブール値ではなくそれらの文字列を探します（多くのプログラミング言語では、 1と 0 は true と falseで同じように使用できますをブール値として使用しますが、 q=my_boolean_fiel d：1で検索しても、my_boolean_ field が true に設定されている動画は返されません）。

    例：このリクエストは、 件名の カスタムフィールドに 動物 という値を持つ動画を返します。

     https://cms.api.brightcove.com/v1/accounts/921483702001/videos?q=subject:animal

    日付範囲

    日付フィールドを検索する場合は、開始日と終了日を区切るために 2 つのピリオドを使用して、特定の日付または日付の範囲を指定できます ( q=updated_at: 2018-01-01.. 2018-02-01 )。

    これにより、2012 年 8 月 1 日から 2012 年 10 月 8 日の間に updated_at の値を持つすべての動画が検索されます。 ここでは、UTC形式で各日付を指定しています。

     q=updated_at: 2012-08-01T 00:00:00 Z.. 2012-10-08T 23:59:59 Z

    時間コンポーネントを削除することで、この検索を簡略化できます。 以下は上記の検索と同等である。

     q=updated_at: 2012-08-01.. 2012-10-08

    サポートされている日付形式

    サポートされている日付形式には、UTC と相対形式があります。

    日付形式の例
    形式 (URI エンコード形式) 意味
    2015-08-01T 06:15:00 Z これは UTC の時刻を表します。
    2012-08-01 これは、UTC の 1 日の午前 0 時を表します。 例は2012-08-01T 00:00:00 Zと同等です
    d 現在の時刻から 1 日を引いた値。 （ 下記 参照）

    相対日付

    相対的な日付では、方向（ + または-）と数字が続き、その後に持続時間が続きます。 相対日付は常に現在の時刻から測定されます。 法的期間は、分、時間、日です。

    + 記号は%2B として URI エンコードされている必要があります 。

    例:

    q=created_at ፦60分.. 60分前から現在時刻に追加された動画

    特定の時刻まですべての日付を一致させる場合、または特定の時刻以降のすべての日付を一致させる場合は、範囲の一端を省略します。

    例:過去 2 日間に変更されたすべての動画を検索する

     q=updated_at ፦2days..

    例:2013 年 8 月 11 日以降に変更されたすべての動画を検索します。

     q=updated_at: 2013-08-11t 00:00:00 Z..

    検索条件を組み合わせる

    複雑な検索の条件を組み合わせることができます。

    例:このリクエストは、2010 年 8 月 1 日から 2010 年 10 月 8 日の間に更新された、 名前 値が ゴシップ の動画を検索します。 次に、 updated_at date で 応答データを降順にソートします。

     q=%2bname: Gossip% 20% 2bupdateD_at: 2010-08-01.. 2010-10-08&sort=-updateD_at

    用語を組み合わせる

    指定された用語 をすべて 含む動画を返すには、 必須/除外する構文を使用します 。

    続く複数の検索 語のすべての例では、追加語に続くテキスト検索語には、最初の語の後に URI エンコードされたスペース (%20) があり、追加語が最初の語の一部として解釈されないように 注意してください。

    たとえば、次のように検索するとします。

     q=名前:foo +タグ:bar (URIエンコード:q=名前:foo% 20% 2BTags: bar)

    応答には、タグ 'bar' を持つ動画が含まれ、名前に foo が含まれる場合もあります。 名前に foo とタグ 'bar'を持つビデオのみを返す場合は、以下を検索する必要があります。

      ( エンコードされていない) q=  +  名前:foo  +  タグ:bar (URI エンコード) q=%2B  名前:foo%20%2B  タグ:bar

    同様に、名前に foo を含むが 'bar' というタグ を持たない 動画だけを返す場合は、次のように検索します。

     （エンコードされていない）q=  +  名前：foo - タグ：バー（エンコードされた）q=%2B 名：foo%20 - タグ：bar

    ステマか

    ステミングはサポートされていますが、部分的な単語検索 はできません 。 たとえば、 q=name: ban は、「 駐車禁止を発表しました」または「禁止 する駐車場」または「都市禁止 駐車場 」という名前の動画を返しますが、「 銀行の休日 」や「 バンディットキャプチャ 」は返しません。

    スペース/特殊文字

    CMS API は、通常、検索文字列の特殊文字を処理しますが、いくつかの例外があります。

    • スペースは使用できません。また、%20 としてエンコードする必要があります 。 （符号化されていない + 記号はスペースを置き換えることもできますが、 + は用語が必要であることを示す可能性があるため、クエリの混乱を招く可能性があります。 「 必須/除外された構文 」を参照）

      たとえば、名前で「お気に入りビデオ」を検索するには、次のようにします。

      q=名前:"my%20お気に入り%20video」

      引用符は、返された動画にすべての用語が含まれていることを保証します。 これらがなければ、3 つの条件のいずれかを含む動画が返されます。 詳細については以下をご覧ください。
    • リテラルの + 記号を検索したり、 + を使用して返される動画に用語を含める 必要 があることを示すには、 + を%2B としてエンコードする必要があります 。

      名前フィールドに 「two+two」 が含まれている必要がある動画を検索する

      q=名前:2%2btwo

      名前フィールドに 「ヘロン」 が含まれている必要がある動画を検索する

      q=%2bname: ヘロン

    • 一部のエージェントはリテラル引用符を正しく処理しない可能性があるため、「foo」 を%22foo%22 としてエンコードする方が安全です
    注意事項:

    複数単語のタグまたは文字列の場合は、タグ/文字列を URI エンコードされた引用符 (%22 ) で囲み、スペースを URI エンコードします ( %20)。

    複数単語タグの例
    エンコードされていない検索文字列 エンコードされた検索文字列
    q=tags:「複数単語タグ」 q=tags: %22multi%20word%20tag%22

    引用符を使用すると、返される動画がすべての用語と一致することを保証しますが、必ずしも正確な用語であるとは限りません。 ステミング はまだ適用されます。たとえば、「新しい家」 で検索すると、 住宅に関するニュースの ような 名前 や 説明 付きのビデオが返されます。

    1 回限りのリクエストでは、Brightcove Learning の 文字列エンコーダ を使用して検索文字列をエンコードできます。 アプリの場合は、使用している言語で URI エンコード機能を見つける必要があります。

    アスタリスク ( * ) は、ワイルドカード ではなく 通常の特殊文字として扱われます。

    検索語をクリップ

    クリップは、他のビデオのセクションから作成されたビデオです。 クリップは Brightcove Social で生成でき、その他の手段も将来利用可能になる予定です。 アカウント内で生成されたクリップを検索するために使用できる特別な検索用語がいくつかあります。

    • Q = %2BIS_クリップ:True -クリップのみを返します。
    • Q=%2BIS_クリップ:false -非クリップのみを返します。
    • q=%2BClip_ソース_ビデオ_ID: video_id -指定したビデオから生成されたクリップを返します。

    無視された単語

    検索文字列では特定の単語は無視されます。なぜなら、検索文字列にはあまり一般的で、実際に検索しているものとは無関係な多くの結果が返される可能性が高いからです。 以下は、検索によって無視される単語のリストです。

    「a」,「an」,「and」,「is」,「as」,「at」,「bat」,「by」,「for」,「if」,「in」,「to」,「is」,「it」,「no」,「ではない」,「の」,「の」,「on」,「or」,「その」,「その」,「彼らの」,「それから」,「そこ」,「これらの」,「彼ら」,「これ」,「へ」,「た」,「意志」,「と」
