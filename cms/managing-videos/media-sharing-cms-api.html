    メディア共有

    このトピックでは、CMS API を使用して、あるVideo Cloud アカウントから別のアカウントへビデオを共有する方法について説明します。

    はじめに

    メディア共有は Video Cloud の機能で、パブリッシャーが他のパブリッシャーとビデオを共有し、複数のアカウントでビデオをより簡単に管理できます。 たとえば、動画コンテンツのマスターアカウントを保持し、組織の他の部門や子会社に動画を共有することができます。

    すべてのメディア共有操作は、Studioでも実行できます。 メディア共有設定を管理 するを参照してください。

    メディア共有は、複数のアカウントを持つ Video Cloud Enterprise および Pro のお客様のみご利用いただけます。 Video Cloud アカウントのアップグレードにご興味のある方は、Brightcove までお問い合わせください。
    CMS API メディア共有操作は、マスターおよび各アフィリエイトアカウントに対して有効にする必要があります。 共有を有効にする必要がある場合は、 ブライトコーブのサポートにお問い合わせください 。

    共有メディアと請求

    共有メディアに対する請求の仕組みについては、「Media モジュールを使用したメディア共有 」を参照してください。

    用語

    メディア共有では、マスターアカウント（動画を共有する）と、共有動画を受信する 1 つ以上のアフィリエイトアカウント（動画を共有する）との間に関係があります。

    メディア共有用語
    アカウント 説明
    マスター 元の動画を作成したアカウント。

    マスターはコンテンツを所有し、アフィリエイトにコンテンツを設定、管理、提供する責任があります。
    アフィリエイト 動画を受信しているアカウント。

    アフィリエイトは、マスターから共有されたコンテンツを受け入れることができます。
    チャネル マスターから任意の数のアフィリエイトにコンテンツを共有するパイプライン。 メディア共有を有効にすると、アカウントに デフォルトの チャンネルが作成されます。
    関係 マスターとアフィリエイト間の相互作用について説明します。

    関係は、コンテンツを共有するマスター、コンテンツを共有するチャンネル、コンテンツを受け入れる契約、コンテンツを受け取るアフィリエイトで構成されています。
    契約 マスターとアフィリエイト間の共有関係について説明します。

    契約はマスターによって作成され、共有を有効にするには、承諾する必要があります。 アフィリエイトは、共有動画を自動的に承認するか、1つずつ承認するかを指定することもできます。

    ベース URL

    すべての CMS API リクエストについては、以下で説明する操作のベース URL は次のとおりです。

     https://cms.api.brightcove.com/v1

    以下で説明するすべてのエンドポイントは、リクエストを行うときにベース URL に追加されます。

    認証

    要求の認証には、認証ヘッダーが必要です。

     承認:ベアラー {access_token}

    access_tok en は一時的な OAuth 2 アクセストークンであり、ブライトコーブ OAuth サービスから取得する必要があります。 クライアントの認証情報を取得し、それらを使用してアクセストークンを取得する方法の詳細については、「 Brightcove OAuth の概要 」を参照してください。

    リレーションシップ に関連するすべての操作には、新しいアクセス許可が必要であることに注意してください。

     ビデオクラウド/ビデオ/すべての
 ビデオクラウド/共有関係/
 ビデオクラウド/共有関係/ビデオクラウド/共有関係
 /ビデオクラウド/共有関係/更新
 ビデオクラウド/共有関係/削除

    あるいは、次のものを使用することもできます。

     ビデオクラウド/共有関係/すべて

    Studio API認証の管理ページには、次の2つの権限が表示されます。

    • 読み取りの共有 （ ビデオクラウド/共有関係/読み取り に相当）
    • 読み取り/書き込みの共有 （ ビデオクラウド/共有関係/すべて に相当）

    共有の制限

    デフォルトでは、すべてのビデオを共有できます。 ただし、次の場合は、共有を禁止できます。

    • アフィリエイトアカウントには、マスターアカウントの動画に値が設定されているカスタムフィールドがありません
    • マスターアカウントではジオフィルタリングが有効になっていますが、アフィリエイトアカウントでは

    カスタムフィールドの一致

    チャンネルにカスタムフィールドのマッチングを適用できます。つまり、 アフィリエイトアカウントに存在しないカスタムフィールドの値が動画に含まれている場合 、動画共有は失敗します。 一致しないカスタムフィールドの値がない動画でも、動画は正常に共有されます。

    既定では、カスタムフィールドの照合は強制 されません 。

    カスタムフィールドが一致しないためにビデオ共有が失敗した場合、応答に次のようなエラーが表示されます。

     { 
「ビデオID」:「5691312273001", 
「アフィリエイトID」:「47509719001", 
「アフィリエイト_ビデオID」: null, 
「ステータス」:「処理中」, 
 「エラー_メッセージ」: [{"エラー_コード」: "ミス_カスタム_フィールド」, "エラー_メッセージ」: "アフィリエイトアカウントにはカスタムがありませんfields: [subject] "}],  
「shared_at」:「2018-01-03T 16:29:19 .080Z」, 
「updated_at」:「2018-01-03T 16:29:19 .080Z」
}

    地理フィルタリングマッチング

    チャンネルで地域フィルタリングマッチングが有効になっている場合、マスターアカウントで地域フィルタリングが有効で、アフィリエイトアカウントが有効でない場合、動画を共有できません。

    共有とは何ですか？

    このセクションでは、共有される内容と、その後のビデオの変更の処理方法について説明します。

    動画が共有されている場合

    ビデオのメタデータフィールドのほとんどは、ビデオが共有されると、マスターからアフィリエイトアカウントにコピーされます。 注目すべき例外は次のとおりです。

    • id -動画はアフィリエイトアカウントに独自の一意のIDを持ちます
    • 日付フィールド ( 作成日時 、 更新日時 )

    すべてのビデオアセット（レンディション、画像、text_tracksなど）は、アフィリエイトアカウントによって再生に使用されます。

    動画が共有された後

    動画が共有された後、マスターアカウントの動画に加えられた変更の一部は、アフィリエイトアカウントに自動的に継承され、一部には反映されません。

    動画アセット

    画像を除き 、動画アセットに対するマスターの変更 は常に アフィリエイトによって継承されます。 アフィリエイトは、レンディション、マニフェスト、テキストトラック、デジタルマスターなどの アセットを変更することはできません 。

    マスターによる画像の変更は、アフィリエイトが 画像を置き換えない限り、アフィリエイトに 継承されます。 アフィリエイトが画像を変更すると、その画像はマスターから継承されなくなります。

    動画メタデータ

    動画メタデータ（名前、説明、参照IDなど）はアフィリエイトによって変更することができ、マスタービデオに加えられた変更はアフィリエイトによって継承 されません 。

    動画の再共有

    ただし、マスターがビデオ を再共有 する場合（これはStudioではなくCMS APIを介してのみ実行できます）、すべてのアセットとメタデータ（データ/時間フィールドを除く）がアフィリエイトに共有され、アフィリエイト が行った変更は上書き されます。

    再共有するには、異なる結果をもたらす2つの方法があることに注意してください。
    ビデオの 共有を解除 し、 再共有 する
    この場合、以前に共有されたビデオは、共有解除のすべてのアフィリエイトアカウントから削除されます。 マスターが動画を再び共有すると、 新しい 一意のIDを持つ 新しい 動画が各アフィリエイトアカウントに追加され、元の共有と同じマスターアカウントのメタデータとアセットが取得されます。
    マスターは、 共有を解除 せずに ビデオを再 共有します
    この場合、動画のマスターコピーからのすべてのメタデータと画像はアフィリエイト版にプッシュされ、アフィリエイトの変更は上書きされますが、アフィリエイト動画IDは変更 されません 。
    動画を再共有する前に、これがやりたいことであり、アフィリエイトに何が起こるかを知っていることを確認してください。

    制限事項

    現在、メディア共有には次の制限があります。

    • DRM: CMS API を介したメディア共有は、現在 DRM が有効なアカウントではサポートされていません。 DRM が有効でないアカウントから DRM が有効なアカウントへの動画の共有はサポートされますが、共有動画は DRM 用にパッケージ化 されません 。

    • マスターアカウントで定義されているチャンネルが enforce_custom_fields を true に設定し、アフィリエイトアカウントで許可されていない値を持つカスタムフィールドを持つ動画を共有すると、その共有は失敗します。 共有ステータスは、次のようなエラーメッセージで更新されます。

       [ {"エラーコード」:「ILEGAL_カスタム_フィールド_値」,「エラーメッセージ」:「カスタムフィールドの無効な値:[トピック]"}]

      マスターアカウントで定義されているチャンネルが enforce_custom_fields を false に設定し、アフィリエイトアカウントで許可されていない値を持つカスタムフィールドを持つ動画を共有すると、共有試行は機能しますが、不適切な値を持つフィールドはビデオのアフィリエイトコピー。

    • SSAIで共有ビデオを再生する場合、SSAIマクロ置換では、子ビデオではなく親ビデオのメタデータが使用されます。 また、親動画が [広告 ] = [無料] とマークされている場合、子動画が [広告サポート] とラベル付けされていても、SSAI は広告 検索をスキップします。
