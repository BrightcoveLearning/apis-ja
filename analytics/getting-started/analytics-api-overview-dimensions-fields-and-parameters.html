    概要:ディメンション、フィールド、およびパラメータ

    ディメンションは、Analytics API データレポートの主要なデータカテゴリです。 このトピックでは、ディメンションとそのディメンションに対して返すことのできるフィールドに関するインタラクティブなガイドを提供します。 また、レポートで結合できるディメンションと、さまざまな組み合わせで使用できるフィールドも表示されます。

    ディメンションとフィールド

    ディメンションは、分析のための主要なデータバケットです。 個々の寸法の完全なガイドを表示するには、下のリストで寸法名をクリックします。

     
     

    下のディメンションを選択して、返すことのできるフィールドを確認します。 [リクエスト を作成] ボタンをクリックして、サンプルリクエスト を作成し、結果を確認することもできます。 互換性のない寸法を複数選択すると、その旨を示すメッセージが表示されます。

    注: video.custom_fields フィールド を返すことはできませんが、場合によっては video.custom_fields を返すこと もできます 。 {フィールド名} 。 これは以下に示されていますが、 {field_name} は実際のフィールド名ではないため、サンプルリクエストでは使用されません。
    入力

    レポートするディメンションを選択:

     
     

    戻すフィールド: ]

     
     

    （サンプルの Brightcove アカウントを使用）

    ad_mode_begin 、 ad_mode_complete 、および play_request のデータは、2016年06-13日から利用可能です。
    出力

    この次元の組合せの最早開始日:

     

    サンプル API リクエスト:

    応答データ

     応答はここに表示されます...

    メモ

    1. デフォルトでは、 video_view だけが返されるフィールドです。その他のフィールドは、fiel ds パラメータの値で指定されている場合にのみ返されます。
    2. ディメンションまたはディメンションの組み合わせでサポートされていない返すフィールドを指定すると、 UNSUPORTED_FIELD_COMBINATION_ERROR エラーが 返されます。
    3. bytes_deliver ed フィールドには、ビデオデータ、画像、テキストトラック、その他のアセットを含む Video Cloud からクライアントに配信されるすべてのデータ、およびプレーヤーコード自体が含まれます。 このデータの一部は CDN から取得され、最大 3 日間利用できない場合があります。
    4. ビデオ ディメンションに表示されるフィールドに加えて、 video.custom_fields も返すことができます。 {フィールド名}

    リクエストの例

    複数のディメンションに関するレポートを取得する一般的なユースケースです。デスクトップとモバイルデバイスの間のビデオビューの内訳を求め、iOS デバイスと Android デバイスでのモバイルデバイスのビューの数や、Mac と Windows マシンでのデスクトップビューの数を調べる必要があります。 現在、Studio Analytics モジュールには、この情報を提供する標準レポートはありませんが、この Analytics API 呼び出しで入手できます。

     https://analytics.api.brightcove.com/v1/data?accounts=57838016001&dimensions=video,device_type,device_os&from=2014-01-01&to=2014-04-01&fields=video_view

    この場合、2014 年 1 月 1 日から 4 月 1 日までの動画視聴をリクエストしています。)

    cURL の使用例

    cURL を使用してAPIを試したい場合は、ここにいくつかの注意事項があります：

    • まず アクセストーク ンを取得する必要があります
    • リクエストの URL には常に URL パラメータ が含まれるため、引用符で囲む必要があります (シングルまたはダブル)。

    サンプル

    次に、cURLコマンドの例を示します。

     curl-s —header「承認:ベアラ $ACCESS_TOKEN "\ 
 "https://analytics.api.brightcove.com/v1/data?accounts=$ACCOUNT_ID&dimensions=video&from=2017-04-04&limit=100"

    $ ACCESS_TOKENを有効なアクセ ストークンに、 $ ACCOUNT_ID をアカウントIDに置き換えると、このリクエストはうまくいくはずです。 このサンプルアプリ を使用してアクセストークンを生成できることに注意してください。

    アカウント

    レポートの対象となる Video Cloud アカウントは、 acc ounts パラメーターを使用して指定します。 たとえば、次のようになります。

     https://analytics.api.brightcove.com/v1/data?accounts={account1_id,account2_id}

    ここで、フィルタ

    フィルタの一般的な構文は次のとおりです。

     ここで、寸法1=値1、寸法2=値2

    たとえば、次のようになります。

     https://analytics.api.brightcove.com/v1?accounts =  アカウントID  &寸法=デバイスタイプ&どこ= ビデオ=ID ; デバイスタイプ==タブレット

    カンマは論理 OR として扱われ、セミコロンは論理 AND として扱われます。 たとえば、 =video==1234,5678; player==9876は「where video = 1234 O R 5678 AND player = 9876" と 解釈される

    スペースと特殊文字

    文字列値は URI エンコードする必要があります。 「」を使用して特殊文字をエスケープすることもできます。

     どこ=検索用語===ボストン、％20mA

    フィルタとして任意のディメンションを使用できます。 ただし、そのディメンションがリクエストする ディメンション にも含まれている場合に 限 ります。

    プレイヤーをフィルタリングする場合は、次の点に注意してください。

    • ブライトコーブのプレイヤー-プレイヤー の URL を次の形式で入力します。 players.brightcove.com/アカウントID/player_id _default
    • プレーヤーを識別するために アプリケーション ID を使用した場合は、プレーヤーの URL を次の形式で入力します。 players.brightcove.com/account_id/player_id_default/ アプリケーション ID (例: players.brightcove.com/1752604059001/bjfywrtul_default/alltimeViewsplayerSample)
    • スマートプレイヤー-プレイヤーIDを入力してください

    ビデオプロパティによるフィルタリング

    特殊な where=video.q== {property}: {value} フィルタを使用すると、次のようなさまざまなプロパティに基づいて、レポートを特定の動画セットに制限できます。

    • タグ
    • reference_id
    • カスタムフィールド [1]
    • {a_specific_field}
    • created_at

    メモ

    [ 1] 基本的な構文は、 =video.q==カスタムフィールド：値 （任意のカスタムフィールドの値と一致する）または =video.q===myfield：値 （特定のカスタムフィールドの値と一致する）です。 特定のカスタムフィールドを検索する場合は、表示名ではなく 内部名で 検索する必要があることに注意してください。

    正しい名前を使用しているかどうかを簡単にチェックできます。内部名は すべて小文字になり、スペースは含まれません 。

    例

    タグとカスタムフィールドを検索するためのフィルタの例を次に示します。

    単一タグ
     どこ=video.q==タグ:foo
    複数のタグ:
     どこ=video.q==タグ:foo, bar
    カスタムフィールド
     どこ=video.q==カスタムフィールド:foo
    タグとユーザー設定フィールド
     ここで、=ビデオ.q==タグ:foo, バー+カスタムフィールド:魚

    このクエリ構文の詳細については、「 CMS API の使用:ビデオの検索 」を参照してください。

    このフィルタを使用して取得したデータは 24 時間キャッシュされるため、過去 1 日に行われた変更は反映されません。

    日付範囲

    すべてのタイプのレポートの from パラメータと to パラメータで指定される日付範囲は、さまざまな形式で指定できます。

    • テキスト値:
      • to=now （すべてのリクエストで使用可能で、デフォルト値）
    • エポック時間の値（ミリ秒単位）（ 1377047323000 など）
    • ISO 8601規格の国際日付形式: YYYY-MM-DD 形式 (2013-09-1 2 など) で表される日付。 この形式で表される日付の場合：
      • 指定された日付範囲は、 アカウントに設定されたタイムゾーンで解釈されます。
      • 日付を与える時刻は、 アカウントに設定されたタイムゾーン で指定された日付の午前0時（ 00:00:00 ）と解釈されます
    • 相対日付： to 値とf ro m値のどちらかをd（日）またはh（時間）で表すことができます。 たとえば、次のようになります。
      • from=2015-01-01&to=31d
      • from=-48h&to=now
      • from =-2d&to=now （前の例と同じ結果が得られます）
      • from=-365d&to=2014-12-31

      負の数（-2d）は「前」（もう一方の値）として解釈され、正の数（48h）は「from」（もう一方の値）として扱われることに注意してください。

    1 日の「ビデオ」のようなディメンションに関するレポートを生成するには、その日付に to 値と from 値を設定します。

     ... &dimensions=video&from=2013-11-01&to=2013-11-01
    注意事項:
    • エンドポイント /data/status を使用して、どのレポートでも利用可能な日付範囲を取得できます。詳細については、 API リファレンスを 参照してください。

    限界とオフセット

    制限 は、返されるアイテムの数です（デフォルト：10）。 すべてのアイテムを返すには、 limit=all を使用します。 offset はスキップする項目の数です (デフォルト:0)。 制限 と オフセット を併用して、結果をページスルーするアプリを作成できます。

    リコンサイルされたデータ

    リコンサイルされた パラメータはブール値です。 true に設定されている場合、結果はリコンサイルされたデータに限定されます。 false の場合、結果はリアルタイム (時間単位の調整なし) データに制限されます。

    リアルタイムデータは、過去 32 日間のみ使用できます。 reconc iled=false に設定し、開始日を 32 日前よりも前に設定した場合、API はデータを返しません。これは、その時点でリコンサイルされていないデータがないためです。

    地理的レポート

    地理的分析のためのディメンション

    • 国 -ISO-3611-1の国コードとして。例えば：「米国」
    • 地域 -ISO-3611-2地域コードとして。例えば：「米国-WA」
    • 都市 -都市名。例えば：シアトル

    注：不明な国または地域の場合、APIはコードとして「ZZ」を返します（ISO-3611-alpha2に従って）。

    フィールドとソート

    f ields パラメーターを使用して、返すフィールドを指定します。 デフォルトでは、 video_view が返され、レポートするディメンション ( destination_domain など) に対応するフィールドが返されます。 詳細については 、ディメンションとフィールド を参照してください。

    sort パラメーターを使用して、返される項目のソートに使用するメトリックスフィールド (例: sort=video_view ) を指定します。 ソートフィールドを無効にしてソート順を逆にすることができます： ソート= - video_view

    返される項目または並べ替えに使用できる項目は、レポート対象のディメンションに対して返される 指標 によって異なります。 これらのパラメータは厳密に検証され、 フィールドまたは ソート パラメータがリクエストに対して無効なフィールド を指している場合、リクエストはエラーを返します。

    ビデオディメンション・レポートの場合、有効なソート値には、ビデオ・メタデータ・フィールド ( ビデオ.name 、 ビデオ.reference_id 、および ビデオ.tags ) が含まれます。

    video_name と video.name

    • video_name は、分析データに記録されるビデオ名です。
    • video.name は、ビデオメタデータ内のビデオ名です。

    ほとんどの場合、2つの値は同一ですが、そうでない場合があります。

    • 場合によっては、 video.name の値が表示されますが、 video_name の値は表示されません。この場合、何らかの理由でビデオ名が分析データコレクターに送信されませんでした。
    • 場合によっては、値が異なる場合があります。この場合、ビデオはある時点で名前が変更されています。 video.name は Vide o Cloud メディアライブラリ内のビデオの現在の名前です。 video_name は、データが分析データに送信された時点のビデオのビデオの名前になります。コレクターを使用します（名前が変更されたピリオドを照会すると、API によって返される値は、その期間中の大部分のビューに関連付けられた名前になります）。

    計算フィールド

    次の構文を使用して API リクエストに計算フィールドを追加できます。

     フィールド=計算フィールド名:式

    計算フィールドを使用して、既存の指標から独自のカスタムフィールドを作成したり、既存のフィールドの名前を変更したりできます。

    計算フィールドを別の計算フィールドの式の一部として使用 することはできません 。

    計算フィールドの名前には、任意の URI 互換文字列を使用できます。 式には、正規フィールド名と次の算術演算子を含めることができます。

    • + (加算)
    • - (減算)
    • * (乗算)
    • / (分割)
    • ^ (指数)
    • () （括弧）

    例

     fields=avg_seconds_viewed: video_seconds_viewed/video_viewed, video.name 
     fields=avg_incomplete_ads: (ad_mode_begin-ad_mode_complete) /video_view, video.name 
     フィールド=ビデオ%20ビューズ:ビデオ_ビュー, ビデオ名
