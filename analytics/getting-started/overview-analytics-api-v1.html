    概要:アナリティクス API v1

    このトピックでは、アナリティクス API の概要について説明します。
     

    はじめに

    Analytics API を使用すると、Video Cloud アカウントの分析データを直接取得できます。 組み込みの分析レポートは、Video Cloud Studio の分析モジュールでも表示できます。 プログラムでデータにアクセスすると、柔軟性が向上します。

    「 API リファレンス 」も参照してください。

    一般的な用途

    APIの一般的な用途は次のとおりです。

    • カスタムチャートとディスプレイの作成
    • 複数の API の連携-CMS API を使用した動画データの取得など、前週の視聴回数の多い動画について
    • 動画分析データを他のサイト分析データと組み合わせる
    • いくつかのサンプルソリューションについては、」
      • プレイリストの分析
      • 国別および日別分析

    ベース URL

    アナリティクス API のベース URL は次のとおりです。

     https://analytics.api.brightcove.com/v1

    ヘッダー

    認証（必須）

    アナリティクス API は、ブライトコーブ OAuth サービス を使用してコールを認証します。

    まず、クライアントの認証情報（ client_id と client_s ecret ）を取得する必要があります。 これは、 OAuth 認証情報 UI を使用して実行できる 1 回限りの操作です。 クライアント認証情報は、CURL、 Postman 、または Insomnia を使用して Bright c ove OAuth サービスから直接取得できます。

    クライアント認証情報には、Analytics の読み取り権限とビデオ読み取り権限の両方が必要です。

    OAuth APIを使用して認証情報を直接作成する場合、必要な権限は次のとおりです。

     [ 
「ビデオクラウド/分析/読む」
「ビデオクラウド/ビデオ/読む」
]

    また、 access_token が必要になります。これは、 client_id とc lient_secret を使用して取得され、APIリクエストとともに認証ヘッダーに渡されます。

     承認:ベアラー  {access_token}

    access_token は 5 分後に期限切れになるので、リクエストごとに 1 つを取得するか、トークンがまだ有効であることを確認する必要があります。 コードサンプルを含む アクセストークンの 取得方法の詳細については、「アクセストークンの取得」を参照してください。

    受け入れエンコーディング：gzip（オプション）

    このヘッダーを渡すと、応答が圧縮形式で返されます。 これにより、大きなレポートのパフォーマンスが向上します。

    キャッシング

    パフォーマンス上の理由から、API レスポンスは約 5 分間キャッシュされますが、正確な時間はいくつかの要因によって異なる場合があります。 Analytics API クエリでは、応答ヘッダーからキャッシュに関する情報を取得できます。

    Cache-Control は、結果がキャッシュされる最大時間を秒単位で示します（上記の例では、24秒）。 Last-Modified ヘッダーと Expires ヘッダーは、現在のキャッシュが作成された日時と有効期限を示します。

    ほとんどの場合、これは問題ではありませんが、分析データの新鮮さが重要な場合は、クエリの実行時間が長ければ長いほどキャッシュされ、リアルタイム（調整されていない時間単位）データを取得するレポートは、調整されたデータを取得するレポート限りキャッシュされません（リアルタイムデータのみ、またはそれに加えて）。 必要に応じて、 リアルタイムおよびリコンサイルされたデータの完全な説明 をご覧ください。短いバージョンでは、 Analytics API が 2 つのデータバケットに依存しています。

    • リアルタイムまたは時間単位の調整されていないデータ。すぐに利用でき、32日間保存されます。
    • リコンサイルされたデータ。このデータは永続的に保存されます。リアルタイムデータは正確性を向上させるためにリコンサイルされ、リコンサイルされたデータリポジトリに 24 時間ごとに格納されます。

    リコンサイルドパラメータを使用して、結果をリコンサ イル またはリアルタイムデータに制限できます。

    キャッシュを最小化する手順は、次のとおりです。

    • recon ciled=false パラメータを使用して、結果をリアルタイムデータに制限します。
    • 小さな 日付範囲を使用し、範囲 全体が過去 32 日以内であることを確認します。

    タイムアウト

    Analytics API リクエストが完了していない場合、8 分後にタイムアウトします。 8 分未満のタイムアウトが表示される場合、原因はクライアント側の制限です。

    返品可能な商品の最大数

    返品可能なアイテムの最大数は100万個です。 ほとんどの場合、制限に達することはほとんどありませんが、たとえば、長い期間にわたって 日付 ディメンションに関するレポートを要求している場合は、可能です。 100万個分の商品制限に達した場合は、リクエストを変更して返品される商品数を減らす必要があります。 一般に、これを行う最も簡単な方法は、データ範囲を縮小することです（後で説明する f rom パラメータと to パラメータを使用します）。

    コンカレント要求

    1 つのアカウントは、一度に 1 つのリクエストに制限されます。 複数の同時要求が連続して実行されます。

    たとえば、次のようになります。

    1. API リクエスト「A」を開始します。
    2. 同じアカウントの API リクエスト「B」を開始します。
    3. 要求「B」は、「A」が完了するまで完了しません。
    4. リクエスト「A」に時間がかかりすぎると、リクエスト「A」に「リクエストは保留中です。もう一度やり直してください」というエラーが表示されます。
    5. 要求「A」に時間がかかりすぎると、要求「B」が同じエラーを受け取ることがあります。 A + B の完了時間がタイムアウト値より大きい場合、要求「B」はエラーになることに注意してください。

    複数の同時リクエストを行う場合、それらは受け取った順序で一度に 1 つずつ処理されます。

    「保留中のエラー」で返されたリクエストは最終的に完了し、キャッシュに保存されます。 これは、同じデータに対する将来の要求は、5分間のキャッシュの有効期限が切れる前に要求が行われた場合に限り、ほぼ瞬時に返されることを意味します。

    システムは、2-4分待って同じ要求を再度実行することで、保留中のエラーを処理する必要があります。

    ベストプラクティス

    上記の情報に基づいて、ベストプラクティスとして次のことを推奨します。
    1. データなしでアプリ内の他の操作を続行できる場合は、 Analytics API に非同期でリクエストを行う
    2. 応答が バックグラウンドでクエリを処理している場合、要求を自動的に再試行する
    3. とにかくキャッシュされたデータを取得するので、5分ごとにリクエストを繰り返さないでください
    4. 複数のリクエストを同時に実行するか、または連続して実行する場合は、実行時間の短いリクエストを遅らせないように、実行時間の長いリクエスト（最も多くのアイテムとデータを返すリクエスト）を最後に作成してみてください

    要求タイプ

    アナリティクス API は、3 つのリクエストタイプを受け入れます

    データ（レポートとも呼ばれます）
    1 つ以上の ディメンション に関するレポート。 レポート要求のエンドポイントは次のとおりです。
     https://analytics.api.brightcove.com/v1/data?accounts={account_id(s)}&dimensions={dimensions}
    エンゲージメントレポート
    過去 32 日以内の期間で利用可能な詳細なエンゲージメントデータ。 詳細については 、エンゲージメントのセクション を参照してください。
    ビデオ情報エンドポイント
    最小限のレイテンシーで提供される特定の分析データ。 詳細については、「 ビデオデータエンドポイント 」を参照してください。

    フィルタと 日付範囲 をレポートに適用できる場所 。 レポートリクエストには、 このドキュメント で詳しく説明されている追加のパラメータを含めることができます。

    ディメンションとフィールド

    ディメンションとフィールドの詳細については、別のドキュメント「 ディメンション、フィールド、およびパラメータの概要 」を参照してください。

    [パラメータ]

    パラメータの詳細については、「 ディメンション、フィールド、パラメータの概要 」という別のドキュメントを参照してください。

    エンゲージメントレポート

    過去 32 日間は、動画の 100 番目の部分ごとの視聴回数（またはアカウントまたはプレーヤーのすべての動画の平均）を示す詳細なエンゲージメントレポートを利用できます。 (過去 32 日以内の日付範囲をリクエストすると、エラーが返されます)。

    エンゲージメントエンドポイントへのリクエストには、いくつかの制限があります。

    • レポートは、単一のアカウント、プレーヤー、またはビデオでのみ使用できます。
    • フィルタがサポートされていない@@ 場所
    • サポートされているパラメータは、 from と to のみです。
    • 日付形式 (MM-DD-YYYY) で開始値と終了値を指定した場合、レポートには完全な日の値が反映されます。
    • 時間単位のエンゲージメントデータが必要な場合は、f rom と to の値を 、レポートする時間内のエポックミリ秒単位で指定する必要があります。

    アカウントエンゲージメント

    視聴した動画のエンゲージメントの平均値を取得するには、エンドポイントを使用します。

     
  https://analytics.api.brightcove.com/v1/engagement/accounts/:account_id

    プレイヤーエンゲージメント

    プレーヤーで視聴されたすべての動画の平均値を取得するには、エンドポイントを使用します。

     
  https://analytics.api.brightcove.com/v1/engagement/accounts/:account_id  /players/: player_id

    ビデオエンゲージメント

    特定の動画のエンゲージメントデータを取得するには、エンドポイントを使用します。

     
  https://analytics.api.brightcove.com/v1/engagement/accounts/:account_id  /videos/: video_id

    ライブ分析

    Analytics API には、時系列またはイベントごとに、Brightcove Live ストリームのアナリティクスを取得するための 2 つのエンドポイントが用意されています。 詳細については、 アナリティクス API リファレンスを 参照してください。
