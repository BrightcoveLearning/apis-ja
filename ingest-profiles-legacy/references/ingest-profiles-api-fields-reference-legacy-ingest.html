    [ 取り込みプロファイル] フィールドのリファレンス

    これは、レガシーインジェストシステムのインジェストプロファイルのフィールドへの参照です。
    このガイドは、従来の取り込みシステムを対象としています。ほとんどのお客様は Dynamic Delivery システムを利用しています。

    プロファイルフィールド

    取得したコンテンツを使用してプロファイルを更新する目的でユーザーがプログラムでプロファイルをプルする場合、ユーザーはプロファイルをサニタイズして、「id「レンディション用ですが、プロファイル用ではありません。」version "は完全に削除する必要があります。
    プロパティ タイプ 説明 デフォルト
    id 文字列 プロファイル ID (読み取り専用) なし
    version 整数 プロファイルのバージョン (読み取り専用) なし
    account_id 整数 Video Cloudアカウント ID なし
    name 文字列 プロファイル名-アカウント内で一意である必要があります なし
    description 文字列 プロファイルの説明 なし
    date_created 整数 (エポックタイムスタンプ (ミリ秒) プロファイルが作成された日付 (読み取り専用) なし
    date_last_modified 整数 (ミリ秒単位のエポックタイムスタンプ) プロファイルが最後に変更された日付 (読み取り専用) なし
    brightcove_standard ブール値 読み取り専用- falseすべてのカスタムプロファイル用 false
    digital_master オブジェクト デジタルマスターをアーカイブするかどうか、およびその方法を定義します。 false
    numRenditions 整数 イメージレンディションを含む、プロファイル内のレンディションの数 (読み取り専用) なし
    renditions 配列 レンディションオブジェクトの配列 []
    packages 配列 DRM パッケージの配列 []

    プロパティ タイプ 説明 デフォルト
    id 文字列 レンディション ID (読み取り専用) なし
    aspect_mode 列挙型 (文字列) ソースとターゲットのアスペクト比の不一致を処理する方法- preserve、stretch、crop、pad-以外の値を選択した場合は注意してくださいpreserve、幅と高さの両方を変更して、目的のアスペクト比を実現できます。 preserve
    audio_bitrate 整数 オーディオビットレート (kbps) ソースと一致
    audio_sample_rate 整数 Hz単位のサンプルレート-（AAC：8000、11025、12000、16000、22050、32000、44100、48000、96000; FLV：11025、22050、および44100） ソースと一致
    max_audio_sample_rate 整数 最大オーディオサンプルレート（Hz）、オーバーライドaudioSampleRate（（AAC：8000、11025、12000、16000、22050、32000、44100、48000、96000; FLV：11025、22050、および44100） ソースと一致

    crf 整数 ビットレート制御設定で、一定のレート係数を設定します。有効な値は0〜51で、値が小さいほど品質が高くなります。 [1-3] なし

    decoder_buffer_size 整数 kbps で、デコーダに供給されるバッファのサイズbitrate_cap なし
    encryption_method 文字列 使用する暗号化方法、例：aes-128 [1-1] なし
    encryption_key_rotation_period 整数 セグメントのセットごとに異なるキーを使用し、この多くのセグメントの後に新しいキーにローテーションします[1-1] null
    fixed_keyframe_interval ブール値 シーンカットのようなものが追加で配置されるのを防ぎますkeyframes false
    forced_keyframe_interval 整数 は X フレームごとにキーフレームを強制しますが、シーンカットなどに追加のキーフレームを許可します。 なし
    forced_keyframe_rate フロート キーフレームレートを強制し、h264のみ、無視される場合forced_keyframe_interval使用されている なし
    format 文字列 ビデオフォーマット- ts、mp4、flv なし

    h264_level フロート ビットレートとマクロブロックを制約します。 [1-6] 計算された
    h264_bframes 整数 連続するBフレームの最大数（0〜16） [1-7] 0
    hls_optimized_ts ブール値 HLS用に最適化された時間セグメント-HLS出力にのみ適用 なし
    keyframe_interval 整数 キーフレーム間の最大フレーム数 250、オーバーライドkeyframe_rate
    keyframe_rate フロート 1 秒あたりの最大キーフレーム数 keyframe_interval 250の
    label 文字列 イメージレンディションのイメージタイプ- posterまたはthumbnail なし
    live_stream ブール値 これがライブビデオストリームになるかどうか なし
    live_dvr_sliding_window_duration 整数 LivedVR 配信用に保持するストリームの所要時間（秒） 300

      2. 許容できる品質
      3. 良い品質。ほとんどのウェブビデオよりも優れています
      4. 素晴らしい品質。優秀に見える

    • [1-5]レベルが低いほどデコードが簡単ですが、レベルが高いほど圧縮率が高くなります。たとえば、iPhone 3gs はベースラインプロファイルのみをサポートします。メインプロファイルとハイプロファイルは圧縮における明確なステップアップであり、ウェブ再生では正常に動作します。ただし、Flash Player の HD ハイプロファイルビデオには注意してください。
