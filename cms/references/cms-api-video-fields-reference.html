    CMS API ビデオフィールドリファレンス

    これは、CMS API によって返されるビデオフィールドへの参照です。

    ビデオフィールド

    ビデオの作成または更新時に設定できるフィールドについては、次のリンクを参照してください。

    • ビデオの作成時に設定できるフィールド
    • ビデオの更新時に設定できるフィールド
    ビデオフィールド
    フィールド [タイプ] 説明
    account 文字列 ビデオクラウドアカウント ID
    keys 文字列 動画に割り当てられた広告キーと値のペアを表す文字列。 キーと値のペアは key=value としてフォーマットされ、アンパサンドで区切られます。 例:「AdKeys」:「カテゴリ=スポーツ&ライブ=真」
    video 文字列 このビデオを生成するためにクリップされたソースビデオの ID。このビデオが別のビデオのクリップでない場合は null 。
    完了 ブール値 処理が完了しているかどうか-注：新しいビデオを作成すると、completeプロパティは自動的に false に設定されます。 ビデオに1つのレンディションが存在するとすぐに、completeプロパティは自動的に true に設定されます
    created 数値、エポックからのミリ秒 ビデオクラウドシステムでビデオが作成されたとき
    created オブジェクト ビデオを作成したユーザーに関する情報（可能な場合）
    cue マップの配列 キューポイントマップの配列
    fields フィールドと値のペアのマップ (文字列) ビデオに設定されたカスタムフィールド値-このビデオの値を持たないカスタムフィールドはこのマップに含まれません。カスタムフィールド値の最大長は 1024 文字です。
    マニフェスト 文字列 DASH マニフェストのシステム ID
    divery 文字列 動画の配信タイプ:
    • d@@ ynamic_origin -動的配信
    • live_origin -ブライトコーブライブからのライブストリーム
    • リモート -資産
    • static_origin -レガシーインジェストシステム
    • 不明 -判定できませんでした（動画に再生可能なレンディションがない可能性があります）
    説明 文字列。古いショートディスクリプションの代わりになります。 ビデオの簡単な説明（最大長：248シングルバイト文字）
    digital 文字列 デジタルマスターのシステム ID
    disabled ブール値 DRM が有効なアカウントの場合、これを true に設定すると、ビデオが DRM パッケージ化されなくなります。 アカウントが DRM 対応でない場合、このフィールドは表示されません。
    期間 数値、ミリ秒 ビデオの再生時間（ミリ秒）
    経済学 文字列。有効な列挙型値のいずれかである必要があります 「AD_SUPPORTED」（デフォルト）または「FREE」
    folder 文字列 動画が属するフォルダの ID
    geo [1-3] プロパティと値のペアのマップ ビデオの地形制限プロパティ
    digital_master ブール値 ビデオに、再トランスコードに使用できるアーカイブ済みマスターがあるかどうか。 このフィールドを使用して動画を再トランスコードできるかどうかをテストする場合、共有動画では信頼できないことに注意してください。 動画がアカウントと共有されている場合は、 has_digital_master = trueとsharing.by_ external_acct = false の両方をテストする必要があります
    ID 文字列 ビデオ ID
    イメージ プロパティと値のペアのマップ ポスターとサムネイル画像に関する情報マップ
    リンク プロパティと値のペアのマップ 関連するリンクプロパティのマップ
    description 文字列 詳細説明 (最大 5000 文字)
    名前 文字列 ビデオの名前（最大長：248シングルバイト文字）
    enabled ブール値 ビデオをオフライン再生に対して有効にするかどうか
    filename 文字列 取り込まれたビデオファイルの名前-リモートアセットビデオの場合は null になります。
    投影法 文字列 360°ビデオのマッピング投影（例：「エクイレクタングラー」）
    published 文字列 ビデオがいつ公開されたかを示すISO-8601日時文字列-このフィールドは書き込み可能であることに注意してください。デフォルトでは、ビデオが最初にアクティブ化された日付に設定されています。
    reference 文字列 ビデオを一意に識別するユーザー指定の ID（150 文字以内）。 ReferenceIDは、別のシステムでこのビデオを識別するための外部キーとして使用できます。 参照 ID には、スペース、カンマ、特殊文字を含めることはできません。
    スケジュール [1-1] プロパティと値のペアのマップ ビデオの可用性を考慮した開始日と終了日時刻のマップ
    共有 プロパティと値のペアのマップ ビデオの共有プロパティのマップ
    manifest_id 文字列 スムーズマニフェストのシステム ID
    manifest_id 文字列 スムーズマニフェストサーバーのシステム ID
    状態 [1-4] 文字列 アクティブ、非アクティブ、保留中、削除済み
    タグ [1-5] タグの配列 (文字列) ビデオに割り当てられたタグの配列
    tracks HTML5スタイルのテキストトラックの配列 ビデオに割り当てられたテキストトラック（WebVTT ファイル）の配列
    updated_at 日時文字列 動画が最後に変更された日時 [1-2]
    updated オブジェクト 動画を最後に更新したユーザーに関する情報（該当する場合）

    メモ

    • [ 1-1] スケジュールされたビデオは、スケジュールされた利用可能期間外には非アクティブ ではない ことに注意してください
    • [ 1-2] ユーザーまたはVideo Cloudシステムによって変更された可能性があります。
    • [ 1-3] Geo 制限は、Video Cloud の有料アドオンです。Geo フィルタリング機能を購入する場合は、アカウントマネージャにお問い合わせください。
    • [ 1-4] の状態 は [アクティブ] または [ 非 アクティブ ] に設定できますが、[ 保留中 ] または [ 削除済み ] に設定 することはできません 。
    • [ 1-5] タグに関する次の制限事項に注意してください。
      • タグは最大 128 文字です。 128文字を超えるタグがある場合、変更を保存しようとするとエラーが表示されます。
      • タグにはカンマを含めることはできません。たとえば、 屋根、屋根裏部屋、前庭の タグはサポートされていません。
      • 動画には合計で1200個のタグしか持たない
      • 大文字と小文字の区別だけが異なるタグ（小文字、大文字）は同じタグとして扱われ、大文字と小文字は自動的に元のタグのものと一致します。大文字と小文字の区別に関する問題は、Video Cloud の外部で処理する必要があります。たとえば、JavaScript メソッドを使用して文字列を変更する
