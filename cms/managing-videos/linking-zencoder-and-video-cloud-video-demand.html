ZencoderとVideo Cloudの統合

オンラインビデオのデプロイに Video Cloud を使用しようとしている Zencoder カスタマーでも、Zencoder のトランスコーディング機能を活用したい Video Cloud のお客様でも、どちらのプラットフォームも利用できます。

このソリューションがVideoCloudで機能するためには、企業アカウント。また、パブリッシャは、自分のVideo Cloudアカウントからリモートアセットをストリームできる必要があります。通常、これは、コンテンツがBrightcove Houseアカウントにないため、コンテンツを保存できるCDN（Akamai、Limelightなど）との何らかの関係が必要になることを意味します。

コンテンツの所有者は、Zencoder APIを使用して、Zencoderにコンテンツをダウンロードしたり、モバイル、Web、コネクティッドTVなど向けのレンディションを作成したります。次に、Video Cloudプレーヤへのストリーミング用にCDNでこのコンテンツをプッシュすることができます。Video Cloudではリモートアセットは、API、Mediaモジュール、またはFTPマニフェストファイルによって作成する必要があります。これにより、Video Cloud Media API を使用してメタデータと CDN 上のリモートアセットへのポインタを取得できます。

Zencoder を使用してビデオをトランスコードする

  1. Zencoderをアプリケーション（CMS、Webアプリなど）と統合します-を参照してください十分に文書化されたAPIおよび統合ライブラリ詳細については。
  2. パブリッシャの動画プレーヤにコンテンツをストリームできる任意の場所を出力場所にできます。通常は、AkamaiやLimelightのようなCDNです。

ビデオクラウドにビデオを追加する

  1. 新しいビデオを作成するビデオクラウドで。
  2. ビデオの作成リクエストに含めることができないメタデータを追加する必要がある場合は、ビデオメタデータを更新する
  3. ビデオをリモートレンディションとして追加する CDNまたは他のリポジトリからのURLを使用して、ビデオに移動します。ほとんどの場合、レンディションを追加 H.264レンディションまたはを追加する操作HSLマニフェストを追加 HLSレンディションを追加します。
