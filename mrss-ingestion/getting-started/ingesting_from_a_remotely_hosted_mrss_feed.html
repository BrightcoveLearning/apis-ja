入門

Media RSS（MRSS）を使用すると、公開した MRSS フィードから自分のシステムからコンテンツをBrightcoveが引き出せることができます。Video Cloud は MRSS フィードからメタデータを取得し、リモートアセットまたはダウンロード（ホストされている場所）を作成し、ビデオおよび関連するアセット（サムネイル、クローズドキャプションファイルなど）を処理します。

リモートでホストされているMRSSフィードからの取り込みのベータ版への参加に関心がある場合は、アカウントマネージャーにお問い合わせください。

MRSS インジェストの概要

MRSS について

メディア RSS (MRSS) は、RSS フィード内のマルチメディアファイルをシンジケートできる RSS の拡張機能です。技術的な詳細については、メディア RSS 仕様を参照してください。

MRSS の設定

担当のアカウントマネージャーに連絡し、MRSS を使用するよう指示してください。MRSS フィードはパブリック URL でホストする必要があります。また、MRSS フィードに含まれる動画とアセットも公開 URL にする必要があります。URL には、http://または https://を含めることができます。

MRSS を使用する

MRSSを使用するには、メタデータやビデオ、その他のファイル（サムネイル、クローズドキャプション）をWebサーバーでホストし、 XMLマニフェストファイルそのコンテンツを参照します。

MRSS フィードは、5 分ごとにBrightcoveによって引き出されます。

MRSS を使用してメタデータのみをプルする

  1. 説明に従ってXMLマニフェストファイルを作成しますXMLマニフェストファイル。メタデータのみの取り込みには、ooyala:remoteassetタグを使用する必要があります。
  2. 指定したMRSSの場所の1つでXMLマニフェストファイルをホストします。ファイル名とパスが、Brightcoveに提供したパスの1つと正確に一致することを確認してください。

MRSS を使用してビデオ、サムネイル、クローズドキャプションファイルをプルする

  1. Web サーバーで 1 つまたは複数のファイル（ビデオ、サムネイル、クローズドキャプションファイル）をホストします。URL はパブリックにアクセス可能である必要があります。
  2. 作成するXMLマニフェストファイル。ファイル内のエントリがサーバー上のファイルを参照していることを確認します。特に、<media:content>要素のhref属性しなければならない次の例のように、サーバー上のファイルへのHTTPURLである。 
    <media:content url="http://mysite.com/upload/lacrosse_70.mov" />
  3. 指定したMRSSの場所の1つでXMLマニフェストファイルをホストします。ファイル名とパスが、Brightcoveに提供したパスの1つと正確に一致することを確認してください。
    注:残念ながら、「クライアントログ」の概念は、MRSS フィードベースの処理には適用されません。フィードが正常に処理されていることを確認する最良の方法は、次のいずれかを使用してアセットの外観を確認することです。 Video Cloud StudioまたはビデオクラウドCMS API。
